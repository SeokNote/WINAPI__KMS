01-04 :  GetMessage 대신 PeekMessage를 쓴다. 
      -> PeekMessage는 메시지가 들어오던 말던 계속 실행이 되는 함수. 
      -> PeekMessage 인자중 하나인 PM_REMOVE는 쌓인 메시지를 삭제하라는 명령어이다.
      -> 왜 삭제할까? 예를 들면 로아에서 겟메시지를 사용하면 QWER누르면 QWER스킬이 나갈때까지 아무런 행동을 못함
      -> 픽 메시지를 사용하게 되면 QWER을 눌러도 Q의 시전시간 후 최근 메시지 행동을 할 수 있음.
      -> 메시지를 예를들면 키를 입력하거나 무슨 이벤트를 발생시키는것.
      -> Float은 오차범위가 있으므로 == 연산을 하면 안된다. 
      -> 싱글톤 : 하나의 객체로 관리하게 하는 패턴
      -> HWND이란 구조체를 typedef한 것으로 무엇을 나타내나면 int* 를 나타낸다 (포인터는 8바이트 정수)
      -> 왜 private 가 아닌 protected? : 상속되었을 때 사용가능하려고 만든 변수여서
      -> 전체적인 구조 디버깅 해보기.
